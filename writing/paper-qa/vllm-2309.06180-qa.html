vLLM：基于 PagedAttention 的大语言模型高效内存管理

SOSP 2023 · 常见提问 Q1–Q7

Q1Q2Q3Q4Q5Q6Q7

Q1：论文要解决什么问题？

现有 LLM serving 为每个请求预留连续 KV cache，但输出长度事先未知，导致预留槽位、内部碎片和外部碎片；并行采样与束搜索还会重复保存共同前缀。论文要在不改变模型结果的前提下，提高 KV 内存有效利用率和可批处理请求数。

Q2：有哪些相关研究，区别在哪里？

Orca 引入迭代级调度，FasterTransformer 提供高性能执行，但它们仍受连续 KV 分配限制。vLLM 与 FlashAttention 的层次不同：后者优化注意力算子的访存，前者解决请求之间、序列生命周期中的 KV 容量管理。后续的 prefix cache、多层 KV cache 和 P/D 解耦系统大多建立在相似块抽象上。

Q3：论文如何解决这个问题？

每个序列拥有逻辑 block table；逻辑块按 token 顺序增长，物理块从统一池按需分配且无需相邻。PagedAttention 内核通过表项找到 K/V。并行采样或束搜索先共享前缀块，写入分叉时才复制最后的共享块；引用计数负责回收。显存不足时，调度器可选择重算或把块换出到 CPU。

\[\operatorname{Attention}(q,K,V)=\operatorname{softmax}\!\left(\frac{qK^{\mathsf T}}{\sqrt d}\right)V\]

PagedAttention 不改变注意力的数学语义；它改变的是 K、V 的物理寻址方式。内核按 block table 取回各物理块，再完成与连续 KV 完全等价的归一化和加权求和。

Q4：论文做了哪些实验，结论可靠吗？

作者比较 vLLM、Orca 与 FasterTransformer，改变模型规模、输入输出长度、采样方式和请求到达率，测量延迟约束下吞吐及内存利用率。结果显示 KV 有效利用率达到 96.3%，吞吐通常提高 2–4×；序列越长、模型越大或解码分支越多，连续分配浪费越严重，因此收益越明显。

Q5：有什么局限与可进一步探索的点？

块表查询和专用内核带来实现复杂度；块大小需要在内部碎片与调度/元数据开销间权衡。原论文主要研究单个 serving engine 内的 KV 管理，没有解决跨节点 KV 放置、SSD/远端层级、语义前缀复用和多租户公平。可继续探索跨层 block placement、预测式换入换出及工作流级缓存价值。

Q6：如何概括论文的主要内容？

vLLM 的核心不是一种新的模型，而是一套 KV cache 虚拟内存：以固定大小块取代每请求连续大区间，用 block table 解耦逻辑顺序与物理位置，再用共享和写时复制消除重复前缀。它让显存从静态预留资源变成可动态复用的块池，因而成为现代 LLM serving 的基础抽象。

Q7：系统的详细主要流程是什么？

请求到达后先 tokenize 并进入调度队列；调度器为 prompt 所需逻辑块分配物理块，worker 执行 prefill 并写入 KV。随后每个 decode iteration 重新组 batch，PagedAttention 按 block table 读取历史 KV，生成的新 token 写入最后一个未满块；块满时再申请新块。分支序列共享只读前缀并在写入时复制。请求完成后引用计数归零的块回到空闲池；内存紧张时先抢占低优先级请求并选择 swap 或 recompute。

Online Scheduling for LLM Inference with KV Cache Constraints：详细论文问答

Q1: 这篇论文试图解决什么问题？

这篇论文试图解决 LLM 在线 inference batching 中 KV cache 持续增长导致的调度可行性和延迟优化问题。每个请求在 decode 过程中逐 token 生成，KV cache 会随输出长度增长。调度器既想让短请求尽快完成，又必须保证批次中的请求未来不会把 GPU memory 撑爆。

困难在于在线性：未来请求何时到达、每个请求输出多长都未知或只能预测。传统 online scheduling 很少考虑这种“已开始请求会持续占用并增长内存”的约束。论文要给出理论基准、不可能性结果和实际可用算法。

Q2: 有哪些相关研究？

相关研究包括：

  • LLM iteration-level batching：ORCA、vLLM 等支持连续 batching，但通常不从理论角度分析 KV 约束。
  • 在线调度理论：competitive ratio、hindsight optimal、shortest processing time 等，为论文建模提供工具。
  • Memory-aware scheduling：传统系统调度考虑内存容量，但 LLM KV cache 的增长与生成过程绑定，形式不同。
  • 输出长度预测：许多 serving 策略依赖预测 token 数，本文也讨论预测误差影响。

论文的独特之处是同时给出 hindsight optimal benchmark、在线算法不可能性和 MC-SF 实用策略。

Q3: 论文如何解决这个问题？

论文分三步解决。

  • Hindsight optimal：假设已知所有未来请求和输出长度，用整数规划描述最优 schedule。这不是实际算法，而是评估在线策略的强基准。
  • 理论结果：证明在任意对抗到达下，确定性在线算法不存在常数 competitive ratio。这说明最坏情况下不能指望简单在线策略总是接近最优。
  • MC-SF 算法：Memory-Constrained Shortest-First 优先继续处理已开始请求，减少浪费；在加入新请求时按预测输出长度从短到长选择，并检查未来 KV cache 可行性，避免启动一个将来无法容纳的 batch。

直觉是：短请求优先有利于降低平均延迟，但必须被 memory feasibility guard 约束。

Q4: 论文做了哪些实验？

实验既有理论对比，也有数值评估。

  • 合成 workload：控制到达率、输出长度分布、KV capacity，观察不同算法与 hindsight optimal 的差距。
  • 真实数据：使用真实或接近真实的 LLM 请求长度分布，评估平均/尾延迟。
  • 基线：FCFS、memory-oblivious shortest-first、不同 batching 策略等。
  • 指标：latency ratio、平均 JCT、tail latency、memory overflow 可行性、预测误差敏感性。
  • 结论：MC-SF 在实际分布上通常比简单 FCFS 或不看内存的短任务优先更稳定，尤其能避免因忽视 KV growth 导致的不可行 schedule。

实验和理论共同说明：KV memory 不是 serving 的附属约束，而是调度模型的一部分。

Q5: 有什么可以进一步探索的点？

进一步探索：

  • 概率竞争分析：在真实到达分布而非对抗模型下，或许可以给出更乐观保证。
  • 鲁棒预测：输出长度预测错误会影响 MC-SF，可使用区间或分位数预测。
  • prefill/decode 分离：将 MC-SF 扩展到 disaggregated serving，需要同时考虑 prefill batch 和 decode KV 增长。
  • 多 GPU placement：KV capacity 在多卡之间分布，调度还需决定请求放哪张卡。
  • 与 cache reuse 结合：prefix/EPIC/LMCache 命中会改变实际 KV 增量和 prefill 成本。

这篇论文适合做 LLM scheduling 理论基线，后续可加更多真实系统约束。

Q6: 总结一下论文的主要内容。

一句话总结：论文把 LLM 在线生成调度形式化为带 KV cache 容量增长约束的在线问题，并提出 MC-SF。

它先定义知道未来的 hindsight optimal，再证明一般对抗场景下在线算法很难有常数竞争比。随后提出 Memory-Constrained Shortest-First：优先已开始请求、短输出优先加入、并始终检查未来 KV 可行性。实验显示该策略在真实式 workload 上有效。主要贡献是把 KV cache constraint 提升为 LLM serving 调度理论的核心约束。

研究阅读补充：这篇适合提供理论警戒线。很多系统论文提出启发式调度，但这篇说明：在对抗在线到达和 KV 增长约束下，想要通用常数竞争比非常困难。因此读它时应关注模型假设和不可能性证明，而不是只看 MC-SF。对本仓库而言，它提示后续研究最好明确 workload 假设：是最坏情况、随机到达、真实 trace，还是可预测 Agent DAG。只有说清假设，算法的理论保证或实验结论才有说服力。
